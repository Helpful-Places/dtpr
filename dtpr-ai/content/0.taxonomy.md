## Authoring editorial overlays

Element and category pages render an optional editorial body when an
authored md file exists at the page's path. To author one, drop a file at:

- `content/taxonomy/elements/<element_id>.md` — appears below the
  Playground on `/taxonomy/elements/<element_id>`.
- `content/taxonomy/categories/<category_id>.md` — appears below the
  elements list on `/taxonomy/categories/<category_id>`.

Required frontmatter:

```yaml
---
navigation: false   # keep the overlay out of the docus left-rail nav
title: My title     # used by Cmd-K search results
---
```

When an overlay md exists, it replaces the synthetic Cmd-K entry for
that route — the authored entry wins on collisions, with terms in the
body indexed for search. Pages with no overlay md still render
completely; the body section is simply omitted.
