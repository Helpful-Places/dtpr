# Introduction

**Digital Trust for Places & Routines (DTPR)** is an open-source communication standard for digital transparency in the built environment. It provides a structured way to describe data-collecting technologies deployed in public and shared spaces.

## What problem does DTPR solve?

As sensors, cameras, and other data-collecting technologies become ubiquitous in public spaces, people have limited visibility into what data is being collected, why, and who has access. DTPR addresses this by providing:

- A **standardized taxonomy** of elements describing technology, purpose, data types, access, and more
- A **machine-readable API** that applications can consume to display transparency information
- **Localization support** for multilingual deployments
- A **composable schema** where elements can be assembled into datachains for specific deployments

## How it works

DTPR organizes transparency information into three layers:

1. **Elements** — Individual descriptions of technologies, purposes, data types, etc.
2. **Categories** — Groupings that organize elements (e.g., "Technology", "Purpose", "Data")
3. **Datachains** — Complete transparency disclosures that combine elements across categories for a specific deployment

The DTPR API serves this taxonomy data, enabling applications to render transparency interfaces for any deployment.
